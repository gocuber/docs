# 类命名规则

　　一般情况下，在创建和命名类的时候要遵循如下规则；类文件完整路径能通过以下代码取得；

strtr($class, array('\\'=>'/', '_'=>'/')) . '.php';
# ['class_map'] 类映射

　　class_map 类映射，该功能会建立一个从类的名字到类文件路径的映射。当自动加载一个文件时，首先检查映射里有没有该类。如果有，对应的文件路径就直接加载了，省掉了进一步的检查。这让类的自动加载变得更直接更快速。事实上所有的 CubePHP 核心类都是这样加载的；
代码： 

$_G['class_map'] = array(
	'Article_List' => '/class/Article/List.php',
	'Models\\Uc_User' => '/models/Uc/User.php',
);
# ['include_path'] 配置

　　include_path 配置，默认情况下 include_path 只包含 SYSTEM_DIR 目录；你也可以添加所需要的目录到 include_path 里；
代码： 

$_G['include_path'] = array(
	SYSTEM_DIR . 'models/',
	SYSTEM_DIR . 'class/',
);

　　这样在 models/ 或 class/ 目录下的类名可以省去 models_ 或 class_，可以使用 namespace 命名空间来解决类重名问题；
代码： 

namespace Models;

class Uc_User{}
# 使用其他自动加载器

　　可以使用自定义的类自动加载器，或者使用 Composer 来管理依赖包，有关 Composer 的自动加载规则，请参考 Composer 文档（英文） 或 Composer 文档（中文）；
