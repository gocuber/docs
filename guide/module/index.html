# 介绍

　　Module 模块是由控制器、视图及模型组成的一个功能模块；实际上模块就是一个应用；一般情况下存放在 app 目录下；完全可以把模块看成是一个独立的应用；

再来看下目录结构： 

/*

/app                             应用目录

|-- default                      默认应用
|-- |-- controllers
|-- |-- views

|-- admin                        后台模块
|-- |-- controllers
|-- |-- views

|-- app1                         其他模块
|-- |-- controllers
|-- |-- views

|-- models                       模型

*/
# 模块配置
['module_list'] 模块列表

　　一般情况下，在开发项目时，会有多个模块；只需修改配置 $_G['module_list'] 就可以了；

代码： 

// 模块列表
$_G['module_list'] = array('default', 'admin', 'cron', 'app1', 'app2');
['module_default'] 默认模块

　　我们访问url的格式是：http://.../模块/控制器/动作/ ；当访问url省略模块时就会加载默认模块；同样修改配置 $_G['module_default']

代码： 

// 默认模块
$_G['module_default'] = 'default'; //默认值是 default 所以为空或没有配置时加载 default 模块
['module_dir'] 模块目录

　　一般情况下模块存放在 app 目录下；当然我们可以修改配置文件，把模块存放到任意想要的目录；

代码： 

// 默认目录 /app/app1/

/*

/app
|-- app1
|-- |-- controllers       控制器
|-- |-- views             视图

*/


// 存放在 /app/a8/ 目录下
$_G['module_dir']['app1'] = APP_DIR . 'a8/';

/*

/app
|-- a8
|-- |-- controllers
|-- |-- views

*/


// 存放在 app 根目录下
$_G['module_dir']['app1'] = APP_DIR;

/*

/app
|-- controllers
|-- views

*/
