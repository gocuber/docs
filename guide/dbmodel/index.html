数据库模型(model)的使用
数据库模型(model)的使用

# 介绍

　　Model 模型 是应用程序中用于处理应用程序数据逻辑的部分；
　　通常模型负责在数据库中存取数据；
　　一般情况下，一个数据库表对应一个 models 类文件；

　　使用 DB::model('user') 来调用models类文件；
　　DB::model('user') 这里的user并不是表名称；而是对应一个models目录中的类文件；
　　注：CubePHP框架中提供一个函数方法 M(); 等同于DB::model(); 所以也可以使用 M('user'); 来调用models类文件；


　　举例说明；

　　库名：ucenter 用户中心库
　　表名：uc_user 用户表
　　表名：uc_group 用户组表

# 创建用户表model

　　使用前缀 uc ，新建 uc 目录用于存放用户中心库下所有的表 models 类文件；
　　新建 models 类 user.php 保存到 models/uc/ 目录中；

　　代码如下： 


	class uc_user extends DB               //继承DB类
	{

		public $_dbname     = 'ucenter';   //数据库名称
		public $_name       = 'uc_user';   //表名称
		public $_primarykey = 'id';        //如果有主键；主键字段名称

		public function getConfig()
		{
			return $GLOBALS['_G']['db']['default'];      //数据库配置；配置文件 config.php
		}

	}

　　接下来就可以 DB::model('uc_user') 这样使用了；注：uc_user 对应的是 models 类名；

# 创建用户组表model

　　新建 group.php 保存到 models/uc/ 目录中；如下：


	class uc_group extends DB
	{

		public $_dbname     = 'ucenter';
		public $_name       = 'uc_group';
		public $_primarykey = 'id';

		public function getConfig()
		{
			return $GLOBALS['_G']['db']['default'];
		}

	}
# 简化写法

　　为了简化写法；可以给每个数据库建立一个类；统一放到 db 目录中；库中所有表继承这个类；如：

　　新建 ucenter.php 保存到 models/db/ 目录中； 


	class db_ucenter extends DB
	{

		public $_dbname     = 'ucenter';
		public function getConfig()
		{
			return array();
		}

	}

　　前面的表 models 类 uc_user uc_group 可以改写成：


	class uc_user extends db_ucenter
	{
		public $_name       = 'uc_user';
		public $_primarykey = 'id';
		// ..
	}

	class uc_group extends db_ucenter
	{
		public $_name       = 'uc_group';
		public $_primarykey = 'id';
		// ..
	}
# model的使用 DB::model()

　　使用 DB::model('uc_user') 直接操作用户表；不用再使用 connect() 方法连库；也不用使用 name() 方法指定表名；如下：


	DB::model('uc_user')->get();

	DB::model('uc_user')->where()->line();

	DB::model('uc_user')->val('name');

	DB::model('uc_user')->delete();

	DB::model('uc_user')->update();

	// ...

　　DB::model('uc_user') 相当于前面的 DB::connect()->name('uc_user')
　　get() line() delete() update() 等方法用法同前面讲的一样；
　　还可以直接使用 M() 函数来调用 model ；如下： 


	M('uc_user')->get();

	M('uc_user')->where()->line();

	M('uc_user')->val('name');

	M('uc_user')->delete();

	M('uc_user')->update();

	// ...
# getFields() 方法

　　在 model 类中可以写 getFields() 方法；来指定表字段类型；如下：


	class Uc_User extends DB_Default
	{
		protected $_name       = 'uc_user';
		protected $_primarykey = 'id';

		public function getFields()
		{
			$fields = array(
				'id',            // 用户id
				'username',      // 账号
				'password',      // 密码
				'status',        // 状态
				'is_sys',        // 是否系统用户
				'is_super',      // 是否超级管理员
				'mobile',        // 手机
				'email',         // email
				'name',          // 姓名
				'nickname',      // 昵称
				'lastlogintime', // 最后登录时间
				'lastloginip',   // 最后登录IP
				'createtime',    // 创建时间
				'updatetime',    // 更新时间
			);
			return $fields;
		}
	}

　　getFields() 方法主要用于 insert() update() 方法中传数组时候会自动处理字段类型；


	$dbdata = array( 'username'=>'user1', 'password'=>'1234567', 'status'=>1, );

	DB::model('uc_user')->insert($dbdata);

	DB::model('uc_user')->where(array( 'id',1 ))->update($dbdata);

　　需要注意的是不要传入 getFields() 方法中不存在的字段；


	$dbdata = array(
		'username'  => 'user1',
		'password'  => '1234567',
		'status'    => 1,
		'groupname' => 'sys',        //这个字段会丢掉，因为 getFields() 方法中没有此字段
	);

	DB::model('uc_user')->insert($dbdata);

	DB::model('uc_user')->where(array( 'id',1 ))->update($dbdata);

　　如果此方法 getFields() 返回一个空值 空数组 或无返回值的时候则不受此限制；没有定义 getFields() 方法的时候同样也不受此限制；
