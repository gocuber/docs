    • # 介绍

    　　Module 模块 是由 控制器、视图 及 模型 组成的一个功能模块；实际上模块就是一个应用；一般情况下存放在 apps 目录下；完全可以把模块看成是一个独立的应用；

    再来看下目录结构： 

    /*

/apps                            应用目录

|-- default                      默认应用
|-- |-- controllers
|-- |-- views

|-- admin                        后台模块
|-- |-- controllers
|-- |-- views

|-- testapp                      其他应用模块
|-- |-- controllers
|-- |-- views


/models                          模型目录

*/
    # 模块配置
    ['module_list'] 模块列表

    　　一般情况下，在开发项目时，会有多个模块；只需修改配置文件 config.php $_G['module_list'] 就可以了；

    代码： 

    //模块列表 一个数组
$_G['module_list'] = array('default','test','admin','cron');
    ['module_default'] 默认模块

    　　我们访问url的格式是：http://.../模块/控制器/动作/ ；当访问url省略模块时就会加载默认模块；同样修改配置文件 config.php

    代码： 

    //默认模块
$_G['module_default'] = 'default'; //默认值是 default 所以为空或没有配置时加载 default 模块
    ['module_dir'] 模块目录

    　　一般情况下模块存放在 apps 目录下；当然我们可以修改配置文件，把模块存放到任意想要的目录；

    代码： 

    //默认目录 /apps/test/

/*

/apps
|-- test
|-- |-- controllers       默认控制器目录
|-- |-- views             默认视图目录

*/


//存放在 /testapp 目录下
$_G['module_dir']['test'] = SYSTEM_DIR.'testapp/';

/*

/testapp
|-- controllers       控制器目录
|-- views             视图目录

*/


//也可以分别设置控制器及视图目录
$_G['module_dir']['test'] = array(
	'c' => SYSTEM_DIR.'controllers/',
	'v' => SYSTEM_DIR.'views/',
);

/*

/controllers       控制器目录
/views             视图目录

*/
